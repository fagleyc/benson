Benson

A private household AI assistant. Single-family install. Local-only.

What it is

Benson is a private, single-tenant household AI assistant built and run by the Fagley family in Colorado Springs, Colorado. It lives on a single on-premises server (an NVIDIA DGX Spark) inside one family's home and is reachable only over that home's local network. There is no public sign-up, no SaaS deployment, no third-party access. It is not a commercial product.

Who uses it

Four people, all members of one household:

  • Casey Fagley — owner and developer
  • Lindsey Fagley — spouse
  • Cole Fagley — child
  • Zander Fagley — child

Each household member can individually grant Benson read access to their own Google account (Gmail read-only and Calendar read+write). Grants are per-user; one member's grant never gives Benson access to another member's data. Anyone in the household can revoke the grant at any time at myaccount.google.com/permissions.

What it does with Google data

Benson uses Google scopes only to answer the household member's own natural-language questions about their own inbox and calendar, and to add calendar events when the same household member explicitly asks Benson to do so. Specifically:

  • gmail.readonly — used to answer queries like "search recent email from the insurance agent" or "did Cole's school send anything today." Benson does not send mail, does not index mail, does not store mail, and does not transmit mail outside the user's home network.
  • calendar.events and calendar.readonly — used to answer scheduling questions ("what's on the calendar tomorrow") and to add events the user explicitly requests ("add a dentist appointment for Lindsey next Friday at 3 PM"). Writes always confirm event details before committing.
  • openid, userinfo.email, userinfo.profile — used only to display the connected Google account's email and name on Benson's admin page so the user can confirm they linked the correct account.

Where the data goes

Nowhere outside the home. Benson runs on a single DGX Spark server in the Fagley basement, connected to one home network. There is no cloud storage, no analytics provider, no telemetry endpoint, no third-party share. Calendar event metadata is cached in RAM for at most 15 minutes to answer scheduling questions; nothing is persisted to disk past that window. Email content fetched per-query is never persisted at all.
