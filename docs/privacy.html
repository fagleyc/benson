Privacy Policy — Benson

Last updated: 2026-05-26

Benson ("the App") is a private, single-tenant household AI assistant run by the Fagley family on a single on-premises server inside one home in Colorado Springs, Colorado. The App is not offered to the public; there is no sign-up, no SaaS endpoint, no commercial distribution. This policy describes what data the App accesses via Google APIs and how it handles that data.

Operator

Casey Fagley · caseyfagley@gmail.com

Data accessed

The App requests the following Google OAuth scopes from each household member who chooses to link their Google account. Linking is voluntary and per-user; one member's link never grants access to another member's data.

  • https://www.googleapis.com/auth/gmail.readonly — read access to the linked user's own Gmail messages.
  • https://www.googleapis.com/auth/calendar.events — read and write access to the linked user's own Google Calendar events.
  • https://www.googleapis.com/auth/calendar.readonly — read-only access to the linked user's calendar list.
  • openid, https://www.googleapis.com/auth/userinfo.email, https://www.googleapis.com/auth/userinfo.profile — the linked user's account email and display name.

How the data is used

Solely to respond to the linked household member's own natural-language questions and explicit commands. Examples:

  • "Search recent email from the insurance agent." — the App issues a Gmail search against the linked user's own inbox and reads matching messages back to that same user.
  • "What's on the calendar tomorrow?" — the App lists the linked user's own calendar events for that date.
  • "Add a dentist appointment for Lindsey next Friday at 3 PM." — the App creates the requested event on the linked user's own Google Calendar after confirming the details with that user.

The linked Google account's name and email address are also displayed on the App's internal admin page so the user can confirm they linked the correct account.

Data retention

  • Gmail messages: never persisted. Each query fetches matching messages live from Google, returns the relevant content to the user, and discards the messages from memory at the end of the request.
  • Calendar events: cached in volatile RAM for at most 15 minutes to allow follow-up questions ("what was the second one?") in the same conversation. Never written to permanent storage. Cache is cleared on server restart or after the 15-minute window.
  • Account email + display name: stored in the App's local OAuth-tokens table on the home server only for as long as the user keeps the account linked. Deleted when the user disconnects.
  • OAuth refresh tokens: stored in the local OAuth-tokens table on the home server. Never transmitted off the server. Deleted when the user disconnects or revokes via Google.

Where the data goes

Nowhere outside the home. The App runs on a single physical server (NVIDIA DGX Spark) inside the operator's home. The server is reachable only on the home local network. The App has no public endpoints, no analytics provider, no telemetry destination, no third-party share. Google API requests originate from the home server's local internet connection and target Google's APIs only.

Data sharing

The App does not share any data accessed via Google APIs with any third party. The App does not share any data between linked household members; each member's data is accessible only to their own queries.

User rights

Each linked household member can revoke the App's access at any time by visiting myaccount.google.com/permissions and removing "Benson." Doing so immediately and permanently terminates the App's ability to access that user's Google data. The user can also disconnect from the App's own admin page, which both deletes the refresh token locally and (best-effort) calls Google's revocation endpoint.

Security

The home server runs Linux with no public-facing services. OAuth refresh tokens are stored in a local PostgreSQL database accessible only to the App's service account. All traffic between the App and Google APIs is HTTPS. The App does not log Gmail content or Calendar event content to disk.

Children

Two of the App's users are children of the operator. Their accounts are linked at the operator's discretion in accordance with the family's own arrangements. The App accesses no special children's data and provides no public-facing service to which children could separately sign up.

Changes to this policy

Material changes will be reflected by updating the "Last updated" date at the top of this page. Because the App is single-tenant, all linked household members will be notified directly by the operator if scopes or retention behaviors change.
