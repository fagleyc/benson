Cameras (iPad-as-Eyeball)

Setup an iPad
1. Trust Benson's HTTPS (one-time per device, ~2 min)

Caddy uses an internal CA. Without trusting it, iOS marks the site "Not Secure" and Safari clears camera permission every session — this is the primary cause of the re-prompt loop.

  1. On the iPad, open Safari and tap: ⬇️ Download cert
  2. "This website is trying to download a configuration profile" → tap Allow.
  3. Open Settings → "Profile Downloaded" appears at the top → tap it.
  4. Tap Install (top right) → enter passcode → Install → Install → Done.
  5. Go to Settings → General → About → Certificate Trust Settings.
  6. Toggle Caddy Local Authority ON → Continue.

Result: "Not Secure" disappears, camera permission persists across sessions.

2. Open the camera page

One-tap launchers (open these on the iPad in that room):

📱 Kitchen Family Room Master Bedroom Office

Or any custom name: https://192.168.0.240/camera/<name>/page. Front camera is the default; append ?facing=environment for back camera.

  1. Grant camera permission once.
  2. Tap Share → Add to Home Screen. Launch via the home-screen icon thereafter — standalone mode keeps permissions sticky.
  3. Wake-lock keeps the screen on while the page is foreground.
  4. The status dot turns green once connected. Benson can now snapshot on demand.
3. Belt-and-suspenders (only if Safari still re-prompts)
  1. Settings → Safari → Camera → Allow.
  2. Always launch from the home-screen icon, not via Safari directly.
