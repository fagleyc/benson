Trust Benson's HTTPS on iPad / iPhone / Mac

Caddy uses an internal certificate authority for HTTPS on the LAN. iOS and macOS don't trust it by default — that's the "Not Secure" warning, and it's also why Safari keeps re-prompting for camera permissions every session. Install this cert as a trusted root and both problems go away.

iPad / iPhone (one-time, ~2 min)

  1. On the device, open Safari and tap this link:

    ⬇️ Download benson-caddy-root.crt

  2. Safari pops up "This website is trying to download a configuration profile" → tap Allow.
  3. Open Settings → "Profile Downloaded" appears at the top → tap it.
  4. Tap Install (top right) → enter passcode → Install → Install again → Done.
  5. Now go to Settings → General → About → Certificate Trust Settings.
  6. Toggle Caddy Local Authority (or similar) ON and tap Continue.

That's it. Reload https://192.168.0.240 — no more "Not Secure" warning, and Safari will now persist camera/mic permissions on the Benson Eye page across restarts.

macOS

  1. Download the cert with the link above.
  2. Open the downloaded benson-caddy-root.crt with Keychain Access (Mac will offer this automatically, or right-click → Open With → Keychain Access).
  3. It lands in the login keychain. Find Caddy Local Authority, double-click it.
  4. Expand Trust, set "When using this certificate" to Always Trust.
  5. Close the window — macOS prompts for your password to confirm.

Windows 10 / 11

Trusts the cert system-wide so Chrome, Edge, and Outlook all stop warning. (Firefox uses its own store — see below.)

  1. On the Windows machine, click this link to download:

    ⬇️ Download benson-caddy-root.crt

  2. In File Explorer, find the downloaded benson-caddy-root.crt → right-click → Install Certificate.
  3. Choose Local Machine (requires admin — best for the whole computer) or Current User (no admin, just your account) → Next.
  4. Select Place all certificates in the following store → Browse → pick Trusted Root Certification Authorities → OK → Next.
  5. Finish. UAC may prompt — accept. You'll see "The import was successful."
  6. Close and reopen any browser tab pointed at https://192.168.0.240 — the lock icon is now clean.

Firefox extra step (it ignores the system store): in Firefox open about:preferences#privacy → scroll to Certificates → View Certificates → Authorities tab → Import → pick benson-caddy-root.crt → check Trust this CA to identify websites → OK.

PowerShell shortcut (admin) — same effect as the GUI install for the local machine: 

Import-Certificate -FilePath "$env:USERPROFILE\Downloads\benson-caddy-root.crt" `
  -CertStoreLocation Cert:\LocalMachine\Root

Belt and suspenders for the iPad camera page

After the cert is trusted, if Safari is still re-prompting for camera permission:

  1. Open Settings → Safari → Camera → set to Allow for "Websites I Visit" or specifically for 192.168.0.240.
  2. On the iPad, open Safari to the camera page, then tap Share → Add to Home Screen. The Home Screen launcher uses standalone mode and gets stickier permissions.
  3. Launch via the home-screen icon (not Safari directly). Permissions persist there much better.

The cert file is also available at /opt/benson/middleware/static/benson-caddy-root.crt on the Spark if you'd rather AirDrop it from a Mac to the iPad.
